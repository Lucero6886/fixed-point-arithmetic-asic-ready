# Project 6.2: SC Decoder N=8 RTL Baseline

## 1. Project Objective

Project 6.2 implements and verifies the RTL baseline for a Successive Cancellation Decoder with code length N=8.

The main objective is to translate the Python golden model from Project 6.1 into synthesizable Verilog RTL and verify the RTL output against golden vectors.

The decoder receives:

```text
LLR[0:7]
frozen_mask[0:7]
```

and produces:

```text
u_hat[0:7]
```

This project establishes the first complete RTL implementation of SC Decoder N=8.

At the end of this project, the learner should understand:

```text
how to map the recursive SC decoding algorithm into Verilog RTL
how to reuse the SC Decoder N=4 block inside the N=8 decoder
how to generate left-branch and right-branch LLRs
how to compute top-level partial sums
how to verify RTL using Python-generated CSV golden vectors
how to debug file-path and dependency issues in a multi-file Verilog project
```

## 7. SC Decoder N=8 RTL Architecture

The SC Decoder N=8 is built recursively from the SC Decoder N=4.

At the top level, the N=8 decoder performs:

```text
1. Compute four left-branch LLRs using f.
2. Decode the left N=4 branch.
3. Compute N=4 partial sums from the left decoded bits.
4. Compute four right-branch LLRs using g.
5. Decode the right N=4 branch.
6. Concatenate left and right decoded bits.
```

This mirrors the Python golden model from Project 6.1.

---

## 8. Top-Level N=8 Schedule

Given input LLRs:

```text
L0, L1, L2, L3, L4, L5, L6, L7
```

### Step 1: Compute Left N=4 LLRs

```text
left0 = f(L0, L4)
left1 = f(L1, L5)
left2 = f(L2, L6)
left3 = f(L3, L7)
```

### Step 2: Decode Left N=4 Branch

```text
u_left = SC_Decode_N4(left0, left1, left2, left3, frozen_mask[0:3])
```

This produces:

```text
u0, u1, u2, u3
```

### Step 3: Generate Top-Level Partial Sums

The partial sums are computed using Polar Encode N=4:

```text
partial0 = u0 ^ u1 ^ u2 ^ u3
partial1 = u1 ^ u3
partial2 = u2 ^ u3
partial3 = u3
```

These partial sums are used as the decision bits for the top-level g operations.

### Step 4: Compute Right N=4 LLRs

```text
right0 = g(L0, L4, partial0)
right1 = g(L1, L5, partial1)
right2 = g(L2, L6, partial2)
right3 = g(L3, L7, partial3)
```

### Step 5: Decode Right N=4 Branch

```text
u_right = SC_Decode_N4(right0, right1, right2, right3, frozen_mask[4:7])
```

This produces:

```text
u4, u5, u6, u7
```

### Step 6: Output

```text
u_hat[0:7] = {u0, u1, u2, u3, u4, u5, u6, u7}
```

Under packed Verilog vector convention, the project uses:

```text
u_hat[0] = u0
u_hat[1] = u1
...
u_hat[7] = u7
```

---

## 9. Frozen-Mask Convention

The RTL must use the same convention as the Python golden model:

```text
frozen_mask[i] = 1 → u_i is frozen and forced to 0
frozen_mask[i] = 0 → u_i is an information bit
```

For each leaf decision:

```text
if frozen_mask[i] = 1:
    u_hat[i] = 0
else:
    u_hat[i] = hard_decision(LLR)
```

The hard-decision rule is:

```text
LLR < 0 → bit 1
LLR >= 0 → bit 0
```

This convention must not be changed between Python, RTL, and testbench.

---

## 10. Bit-Ordering Convention

The project uses LSB-first bit indexing.

The CSV file stores:

```text
u_hat0,u_hat1,u_hat2,u_hat3,u_hat4,u_hat5,u_hat6,u_hat7
```

The RTL output vector should match:

```text
u_hat[0] = u_hat0
u_hat[1] = u_hat1
u_hat[2] = u_hat2
u_hat[3] = u_hat3
u_hat[4] = u_hat4
u_hat[5] = u_hat5
u_hat[6] = u_hat6
u_hat[7] = u_hat7
```

The integer representation is:

```text
u_hat_int = u_hat0 * 2^0
          + u_hat1 * 2^1
          + ...
          + u_hat7 * 2^7
```

Any mismatch between LSB-first and MSB-first interpretation will cause verification errors.

## 11. Example RTL Structure

A simplified RTL structure is:

```verilog
module sc_decoder_n8 #(
    parameter W = 6
)(
    input  wire signed [W-1:0] llr0,
    input  wire signed [W-1:0] llr1,
    input  wire signed [W-1:0] llr2,
    input  wire signed [W-1:0] llr3,
    input  wire signed [W-1:0] llr4,
    input  wire signed [W-1:0] llr5,
    input  wire signed [W-1:0] llr6,
    input  wire signed [W-1:0] llr7,
    input  wire        [7:0]   frozen_mask,
    output wire        [7:0]   u_hat
);

    // 1. top-level f operations
    // 2. left N=4 decoder
    // 3. partial-sum generation
    // 4. top-level g operations
    // 5. right N=4 decoder
    // 6. output concatenation

endmodule
```

The actual RTL should instantiate:

```text
SC f operations
SC g operations
left sc_decoder_n4
right sc_decoder_n4
partial-sum XOR logic
```

## 12. Top-Level Partial-Sum RTL

The key partial-sum logic is:

```verilog
assign p0 = u_left[0] ^ u_left[1] ^ u_left[2] ^ u_left[3];
assign p1 = u_left[1] ^ u_left[3];
assign p2 = u_left[2] ^ u_left[3];
assign p3 = u_left[3];
```

These partial sums are then used by the top-level g operations.

This is a critical part of the N=8 decoder.

If this block is wrong, the right N=4 branch will be decoded incorrectly.

## 21. Common Debug Issue 4: Wrong Partial Sums

For the N=8 top-level right branch, the partial sums must be:

```text
p0 = u0 ^ u1 ^ u2 ^ u3
p1 = u1 ^ u3
p2 = u2 ^ u3
p3 = u3
```

A common mistake is to use:

```text
p0 = u0
p1 = u1
p2 = u2
p3 = u3
```

This is wrong for the recursive Polar partial-sum convention.

Wrong partial sums usually cause many vector mismatches.

---

## 22. Common Debug Issue 5: Wrong g Function Order

The correct g function is:

```text
g(a,b,u) = b + a, if u = 0
g(a,b,u) = b - a, if u = 1
```

A common mistake is:

```text
a - b
```

instead of:

```text
b - a
```

This error may not be obvious in simple cases but will fail golden-vector verification.

---

## 23. Common Debug Issue 6: Frozen-Mask Reversal

Correct convention:

```text
frozen_mask[i] = 1 → force u_i = 0
frozen_mask[i] = 0 → hard decision
```

If reversed, the decoder will produce many wrong outputs.

Always check the convention in:

```text
Python golden model
Verilog RTL
Verilog testbench
documentation
```

## 26. Difference Between SC Decoder N=4 And N=8

The N=4 decoder directly implements one small SC tree.

The N=8 decoder uses the N=4 decoder recursively.

N=8 introduces:

```text
top-level f stage
left N=4 decoder
N=4 partial-sum generation
top-level g stage
right N=4 decoder
larger frozen mask
larger u_hat output
```

Therefore, N=8 is the first architecture where recursive composition becomes clear.
